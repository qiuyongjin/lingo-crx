# System Prompt 设计备忘

## 核心理念

让英语初学者**理解并记住**目标单词，而非仅仅获取释义。

## 关键设计决策

### 1. 场景驱动，而非词典罗列

坏场景是"常见于日常对话"——说了等于没说。
好场景包含**具体的人、事、情绪**，能让人脑子里产生画面。

```
坏：apple - 苹果。常见于水果店和超市。
好：apple - 苹果。你咬一口红富士，嘎嘣脆、满嘴甜汁——就是这种感觉。
```

### 2. 宽度低于精度

受众锚定明确（英语初学者），不追求覆盖所有词义，只输出当前语境或最常见用法。

### 3. 自然口语，而非学术腔

"像给朋友随口讲一个词"——用对话语气而非词典定义式的书面语。

### 4. 有硬边界也有弹性

- **硬边界**：最多 3 句话、不输出 markdown 标记
- **弹性**：固定搭配可保留英文（如 "take a nap"），避免强行全中文反而别扭

### 5. 两种模式统一结构

| 模式 | 格式 |
|------|------|
| 仅单词 | 释义 → 场景 |
| 有句子（`{}` 标记目标词） | 语境释义 → 类比场景 → （可选）原句回顾 |

### 6. 反例比正例更高效

一个"坏场景"示例比三行正向描述更能约束模型行为，用最少 token 防住最常见问题。

## 技术配合

CSS 侧 `.lingo-meaning` 需设置 `white-space: pre-line`，否则 AI 输出中的 `\n` 换行会被折叠，排版结构失效。
